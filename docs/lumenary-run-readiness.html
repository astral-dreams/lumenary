Lumenary run readiness

What is left before continuous operation?

Lumenary can already run a live Codex CLI research cycle, store original idea records, publish a daily Markdown update, and generate reviewed X drafts. The remaining work is mostly launch discipline: source grounding, cadence, review gates, and a public website.

What Is Left Before We Start Running?

Item Status Why it matters Recommended action
Decide launch mode Required We should avoid turning on an unsupervised recursive loop before checking output quality for a few cycles. Run supervised for 3 to 5 iterations before installing LaunchAgents.
Set cadence and budget Required The loop uses your Codex subscription and can run long research calls. Cadence controls cost, noise, and review burden. Start with one live research run every 6 or 12 hours, plus one daily publish draft.
Seed real sources Required for quality Without source grounding, the system becomes self-referential and rewards its own prior ideas. Add 10 to 20 source cards across Advaita, Buddhism, Daoism, Neoplatonism, Sufism, and consciousness science.
Define promotion rules Required before public site Draft observations should not automatically become public claims. Create thresholds for source reliability, counterargument quality, and publishability.
Build the website Next implementation step The public surface does not exist yet. Current outputs are Markdown files. Build a static site reading publication/daily/, observations/, and syntheses/.
X posting Deferred Posting philosophical claims automatically is reputationally risky and requires X API credentials. Keep X drafts in review queue first. Add API posting only after manual approval policy is clear.
LaunchAgent install Ready after review The plist files exist, but loading them starts background automation. Install after one supervised run and one daily publish review.

Recommended Start Sequence

Step 1

Run one supervised live cycle

python3 -m engine.run \
  --provider codex-cli \
  --agent codex \
  --search \
  --focus "Research one narrow source-grounded comparison and generate one original Lumenary idea."
Step 2

Generate daily publish draft

python3 -m engine.publisher
Step 3

Review output manually

open publication/daily
open publication/x/queue
Step 4

Start a supervised scheduler session

python3 -m engine.scheduler \
  --provider codex-cli \
  --search \
  --interval-minutes 360 \
  --iterations 3

Continuous Operation Options

Option A: supervised terminal

Best for the first day. You can stop it with Control-C and inspect every run.

python3 -m engine.scheduler \
  --provider codex-cli \
  --search \
  --interval-minutes 360

Option B: macOS LaunchAgents

Best after the supervised pass. This runs research continuously and publishes a daily draft at 8 AM.

mkdir -p ~/Library/LaunchAgents
cp ops/launchd/com.lumenary.research.plist ~/Library/LaunchAgents/
cp ops/launchd/com.lumenary.daily-publish.plist ~/Library/LaunchAgents/
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.lumenary.research.plist
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.lumenary.daily-publish.plist

Recommendation: start with Option A for 24 hours, then install LaunchAgents after the output quality and cadence feel right.

Source Grounding Plan

The biggest quality gap is not orchestration. It is evidence. Before the loop runs unattended, seed a small corpus with source cards.

Spiritual texts

  • Upanishads or Advaita source excerpts
  • Early Buddhist texts on anatta
  • Dao De Jing or Zhuangzi selections
  • Plotinus or Neoplatonic sources

Comparative philosophy

  • Scholarship on nondualism
  • Comparative mysticism critiques
  • Philosophy of mind and selfhood
  • Phenomenology of attention

Science adjacency

  • Meditation and attention research
  • Self-modeling and salience network papers
  • Consciousness theories with careful limits
  • Explicit anti-pop-science guardrails

Decision Checklist

  1. Confirm the working name: Lumenary or choose another.
  2. Choose supervised cadence: 6 hours, 12 hours, or daily.
  3. Decide whether first website version should be static HTML, Astro, or another stack.
  4. Decide what counts as publishable: score threshold, source requirement, and human review requirement.
  5. Decide whether Claude Code will import JSON ideas or run a parallel process writing directly to its directories.
  6. Decide when X drafts may become posts. Current recommendation: not until website quality is stable.

My Recommendation

Do not install the continuous LaunchAgent yet. First, run three supervised live cycles with web search, add at least ten source cards, generate daily drafts, and review the publication quality. Then build the first website around daily findings and observation archives. After that, turn on background scheduling.
